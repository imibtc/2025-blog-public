## 💻 本地 (Local) vs. 远程 (Remote)

*   **本地 (Local)**：指数据或程序仅存储在你的个人设备上（如手机、电脑），不依赖网络。例如，手机离线时编辑的文档只存在于该手机上。
*   **远程 (Remote)**：指数据或程序存储在云端服务器上，通过网络访问。例如，将文档保存到网盘后，便可在任何联网设备上访问。

> **核心要点**：理解代码、文件或模型是在你本机运行，还是在云端运行，是后续所有操作的基础。

## ⌨️ 终端 (Terminal)、目录 (Directory) 与路径 (Path)

###  1. 终端 (Terminal)
终端是一个通过输入文本命令来操作电脑的程序，可以创建、编辑、移动文件等。使用AI编程代理时，你通常只需执行一个关键命令：

```
cd /Users/bentossell/project/bensbites/
```


该命令的作用是**切换目录 (Change Directory)**，即进入项目所在的文件夹，以便AI在该特定环境中工作。

### 2. 目录与路径 (Directory & Path)
*   **目录 (Directory)**：即我们常说的“文件夹”。
*   **路径 (Path)**：是文件或文件夹在电脑中的具体“地址”，用于精确定位。例如，上述 `cd` 命令中的一长串字符就是一条路径。

路径中的 `~` (波浪号) 是一个快捷方式，代表你的用户主目录（如 `/Users/你的名字`）。

> **核心要点**：你无需精通所有终端命令，但必须理解**目录**和**路径**的概念，并会使用 `cd` 命令进入正确的项目文件夹。

## 🤖 AI 编程代理 (Coding Agents) 是什么？

你可以将AI编程代理理解为“会写代码的ChatGPT”。它基于大语言模型（LLM），既能与你对话，又能调用工具（如读写文件、运行命令）来完成任务。

---

## 📜 系统提示词 (System Prompts)

每个AI代理都内置了一套“系统提示词”，它定义了代理的身份、能力和行为边界。例如，它会被告知：

*   你是一个专家软件工程师。
*   你可以使用哪些工具，不应做什么。
*   你的知识截止日期，以及当前日期。

一些公司会公开其系统提示词，你也可以在社交媒体上找到一些“套话”技巧来窥探它们。系统提示词还可能包含联网搜索、查阅自身文档等工具的说明。

---

## 📝 Markdown 文件与 agents.md

### 1. Markdown 文件 (.md)
Markdown是一种带简单格式（如标题、列表）的纯文本文件格式，常用于编写文档。

### 2. agents.md 文件
这是你为AI代理提供的额外指令文件，用于补充系统提示词。不同代理的配置文件名可能不同（如Claude使用 `CLAUDE.md`）。

你可以在 `agents.md` 中写入项目专属指令，例如：
*   **解释风格**：“向我解释代码时，请使用通俗易懂的语言。”
*   **技术栈说明**：“本项目使用React + Vite。”
*   **关键文件**：“入口文件是 `src/App.tsx`。”

**文件查找优先级**：代理会优先读取离目标文件最近的 `agents.md`。查找顺序如下：
1.  `my-project/src/components/Button.tsx` 文件旁的 `agents.md`
2.  `my-project/src/` 目录下的 `agents.md`
3.  `my-project/` 根目录下的 `agents.md`

## 🛠️ 技能 (Skills)

技能是可复用的“工作流”或“模板”，以文件形式存在（如 `skill.md`）。当你需要执行重复性任务时，只需调用对应的技能即可。

**示例：撰写新闻稿标题**
1.  **指令**：你将新闻稿内容发给代理，并说：“根据内容写一个标题。”
2.  **技能文件 (`skill.md`)**：其中包含了写标题的规则，如“不使用表情符号”、“标题不超过4个词”、“必须包含新闻稿的核心关键词”等。

技能还可以与工具绑定，例如一个“写标题+配图”的技能，会先调用写标题的指令，再调用图像生成工具。

---

## 💡 前置说明 (Front Matter) 与渐进式披露 (Progressive Disclosure)

#### 1. 前置说明 (Front Matter)
这是写在文档顶部的元数据（metadata），用于描述文档本身。在技能文件中，它通常长这样：

```
name: Newsletter-content-writer
description: Use this skill when asked to write any newsletter content.
```

### 2. 渐进式披露 (Progressive Disclosure)
指代理如何逐步加载和使用信息。它会按以下顺序处理上下文：
1.  **系统提示词**：了解自身角色和能力。
2.  **agents.md**：加载项目通用指令。
3.  **技能 (Skills)**：根据Front Matter的描述，判断是否需要调用特定技能来处理当前任务。

这种方式避免了信息过载，让代理能更专注地处理任务。

## 📍 技能与配置文件的存放位置

许多AI代理的配置和技能文件是隐藏的，存储在用户主目录下的“点文件”（dotfiles）中。

*   **查看方法**：在Mac上，按 `Command + Shift + .` 即可显示隐藏文件。
*   **典型路径**：
    *   `~/.factory/skills/`：存放技能文件。
    *   `~/.factory/` 目录下也可能有 `agents.md` 或 `commands/`。

建议将核心指令和技能集中存放在这些位置，便于代理统一读取。

---

## ⚡️ 自定义命令 (Commands)

你可以创建自定义命令（如 `/newsletter`）来一键执行复杂的工作流。代理可以帮你自动创建这些命令，你也可以手动在 `~/.factory/commands/` 目录下配置。

## 🐧 Bash：终端的核心工具

Bash是终端中最核心的工具，AI代理通过它来执行各种命令，如查看文件 (`ls`)、创建目录 (`mkdir`)、安装依赖等。你无需记住所有命令，只需理解代理正在通过Bash与你的电脑交互即可。

## 🌿 Git 与版本控制

Git是管理代码版本的核心工具，理解它对于使用AI编程代理至关重要。

*   **提交 (Commit)**：将当前代码状态保存为一个“快照”，并附上说明（如“修复了登录bug”）。
*   **分支 (Branch)**：创建一个项目的平行版本，用于开发新功能而不影响主版本。
*   **本地与远程 (Local & Remote)**：
    *   **本地提交**：仅保存在你的电脑上。
    *   **推送 (Push)**：将本地提交上传到远程仓库（如GitHub）。
    *   **拉取 (Pull)**：从远程仓库下载最新的提交到你的电脑。
*   **仓库 (Repository)**：一个包含 `.git` 隐藏文件夹的项目目录，用于追踪所有版本历史。

AI代理通常会自动处理 `commit`、`push`、`pull` 和 `branch` 等操作。

## 🔑 环境变量与 API 密钥

API密钥是连接第三方服务（如OpenAI、Stripe）的“密码”，**绝不能**直接写在代码中，否则一旦代码被公开，密钥也会泄露。

**正确做法**：
1.  在项目根目录创建一个 `.env` 文件。
2.  将密钥写入其中：
    env
    OPENAI_API_KEY=sk-xxxxxxxx
    
3.  在代码中通过 `process.env.OPENAI_API_KEY` 的方式读取密钥。
4.  将 `.env` 文件添加到 `.gitignore` 中，确保它不会被上传到GitHub。

AI代理可以帮你创建 `.env` 文件和编写读取逻辑，但**密钥本身必须由你提供**。

## 📦 依赖与包管理器

现代软件开发大量使用他人已写好的代码库，这些库被称为**依赖 (Dependencies)** 或**包 (Packages)**。

*   **包管理器 (Package Manager)**：用于安装、更新和管理这些依赖的工具。
    *   **JavaScript/前端**：`npm`, `yarn`, `bun`
    *   **Python**：`pip`

当你克隆一个项目后，AI代理通常会自动运行 `npm install` 或 `pip install` 来安装所有必需的依赖。

---

## 🚀 本地运行项目

项目依赖安装完毕后，就可以在本地运行了。

1.  **启动命令**：通常运行 `npm run dev` 或 `npm start`。
2.  **本地服务器**：命令执行后，项目会运行在 `localhost:3000` 这样的地址上。`localhost` 指你的电脑，`3000` 是端口号。
3.  **热重载 (Hot Reloading)**：在开发模式下，修改代码后页面会自动刷新，无需手动重启服务。

---

## 🌍 部署项目

本地运行成功后，需要将项目部署到线上服务器，让所有人都能访问。

*   **常用平台**：Vercel, Netlify, Cloudflare Pages。
*   **部署方式**：这些平台通常提供命令行工具 (CLI)，AI代理可以调用它们来完成部署。
*   **环境变量**：部署时，你需要在平台的控制面板中重新设置所有必要的环境变量（如API密钥），因为 `.env` 文件不会被部署。

---

## 🧠 上下文窗口与 Token

AI模型处理信息的能力有限，这个限制称为**上下文窗口 (Context Window)**，以 **Token** 为单位（1个Token约等于4个英文字符或3/4个单词）。

如果项目文件过多或指令过长，会挤占上下文空间，导致模型“忘记”之前的信息或难以保持专注。因此，保持 `agents.md` 和技能文件的简洁、只加载必要信息至关重要。

## 🐛 阅读与修复错误

当代码出错时，终端或浏览器控制台会显示错误信息（通常是红色文字）。错误信息会指出问题所在（如“找不到模块”、“未定义的函数”）和发生位置（文件路径和行号）。

你无需精通错误分析，只需将完整的错误信息复制给AI代理，并说明“请帮我修复”，它通常能快速定位并解决问题。

---

## 🔍 浏览器开发者工具 (DevTools)

对于网页项目，浏览器的开发者工具是调试利器。

*   **打开方式**：在网页上右键点击“检查” (Inspect)，或使用快捷键（Mac: `Command + Option + I`）。
*   **Console (控制台)**：查看JavaScript错误和网络请求失败信息。
*   **Network (网络)**：监控所有网络请求，检查API调用是否成功、返回了什么数据。

你可以将控制台的错误信息复制给AI，让它帮你排查问题。一些AI代理（如 `agent-browser`）甚至可以自动帮你操作浏览器并读取错误信息。

## 🔄 Bug修复与测试

遇到Bug时，高效的流程是：

1.  **提供上下文**：向AI说明你正在做什么、期望发生什么、实际发生了什么，并附上错误信息。
2.  **先分析，后修复**：要求AI先解释问题原因，而不是直接修改代码。
3.  **编写测试**：让AI编写一个测试用例来复现Bug。修复Bug后，运行测试以确保问题已解决且未引入新问题。这被称为**测试驱动开发 (TDD)**。
4.  **添加日志**：让AI在关键位置添加日志输出，帮助你和AI追踪代码执行流程。

---

## 🔄 避免无效循环

当修复陷入僵局时，可以尝试以下方法：

1.  **提交代码**：在继续调试前，先 `git commit` 保存当前进度，以便随时回滚。
2.  **开启新会话**：有时旧的会话上下文混乱，开启一个新的AI对话可能会更有效。
3.  **检查配置**：确认 `.env` 文件和环境变量是否正确设置。
4.  **更新指令**：如果AI反复犯同样的错误，说明指令不够清晰。可以问AI：“我该如何修改 `agents.md` 文件，以避免这个问题？”

## 🔄 代理的工作循环 (Agent Loop)

AI代理的工作模式是一个循环：

1.  **计划 (Plan)**：分析任务，制定实现步骤。
2.  **行动 (Act)**：执行计划，如创建文件、编写代码、运行命令。
3.  **观察 (Observe)**：检查行动结果，看是否成功或出错。
4.  **重复**：根据观察结果，决定是继续下一步还是返回修复问题。

---

## 🤖 代理的自主性 (Autonomy Modes)

不同的AI代理提供不同的自主性级别：

*   **低自主性**：每执行一步操作前都会征求你的同意，安全但速度慢。
*   **高自主性**：直接执行操作，速度快，但需要你信任它并随时准备干预。

大多数代理允许你自定义自主性级别，例如“删除文件前需确认，但创建文件无需确认”。

## ✨ 总结：如何开始？

你无需记住所有概念。最好的学习方式是**动手实践**：

1.  选择一个简单的项目（如个人网站、待办清单）。
2.  让AI代理引导你完成从创建项目、本地运行到部署上线的全过程。
3.  遇到不懂的概念，随时回来查阅这份指南。

在实践中，你会自然而然地掌握这些基础知识。
